LOOK Disk Scheduling Algorithm

This algorithm is similar to SCAN algorithm except for the end-to-end reach of each sweep. The R/W head is only tasked to go the farthest location in need of servicing. This is also a directional algorithm, as soon as it is done with the last request in one direction it then sweeps in the other direction

The LOOK algorithm is an improved disk scheduling algorithm that is a variant of the SCAN algorithm. It moves the disk head in one direction until it reaches the last request in that direction, then reverses its direction without going to the end of the disk.

Working of LOOK Algorithm

  • The disk head moves in a specific direction (left or right).
  • It serves all the requests in that direction until it reaches the last request (the farthest request in that direction).
  • Unlike SCAN, it does not go to the end of the disk if there are no requests in that region.
  • After serving the last request in that direction, it reverses and moves in the opposite direction to handle the remaining requests.

Advantages of LOOK Algorithm

  • It reduces unnecessary movements compared to SCAN.
  • It provides better performance and reduced seek time.
  • This algorithm is efficient for large disk sizes with sparse requests.

Disadvantages of LOOK Algorithm

  • It may still suffer from the Starvation problem if new requests keep arriving on the other side.
  • This algorithm is not optimal as more complex algorithms like C-LOOK or SSTF in certain scenarios.
