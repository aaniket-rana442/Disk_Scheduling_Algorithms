Circular LOOK (C-LOOK) Algorithm

Circular LOOK is like a C-SCAN which uses a return sweep before processing a set of disk requests [2]. It does not reach the end of the tracks unless there is a request, either read or write on such disk location similar with the LOOK algorithm.

As LOOK is similar to the SCAN algorithm, in a similar way, C-LOOK is similar to the C-SCAN disk scheduling algorithm. In C-LOOK, the disk arm in spite of going to the end goes only to the last request to be serviced in front of the head and then from there goes to the other end's last request. Thus, it also prevents the extra delay which occurred due to unnecessary traversal to the end of the disk.

Working of C-LOOK Algorithm

  • The disk head moves in one direction (typically from low to high) to serve all pending requests.
  • It stops at the last request in that direction, without going to the end of the disk.
  • Instead of reversing, the disk head jumps back to the lowest request and continues servicing requests in the same direction.
  • This movement forms a circular path, hence the name C-LOOK.

Advantages of C-LOOK Algorithm

  • The circular nature reduces unnecessary movements compared to SCAN or LOOK.
  • It does not go to the end of the disk unless necessary.
  • This algorithm works well with large disk sizes and heavy loads.
  • It provides a more predictable and faster response time for requests.

Disadvantages of C-LOOK Algorithm

  • Requests closer to the starting point may have to wait longer.
  • Slightly more complex to implement compared to FCFS or LOOK.
  • Starvation may occur if continuous requests arrive in one direction.
